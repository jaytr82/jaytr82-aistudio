# Contributing

## Workflow

1. Start from `main`.
2. Create a focused branch for one change set.
3. Keep prompts, standards, skills, and templates aligned when changing repository behavior.
4. Update documentation when the workflow or scaffold changes.
5. Run the relevant checks before opening a pull request.

## Change Expectations

- Keep skills concise and task-specific.
- Prefer extending existing templates over creating duplicate starters.
- Preserve separation between prompts, standards, workflows, and templates.
- Avoid adding dependencies to templates without a clear reason.

## Pull Requests

Include:

1. what changed
2. why it changed
3. files or templates affected
4. validation performed
5. follow-up work if something is intentionally incomplete

## Validation

- For Node templates, run lint and tests.
- For Python templates, run formatting and tests if added.
- For skill changes, verify the trigger scope is still clear and non-overlapping.
