# Architecture Design Prompt

Design a production-grade software architecture for the following product:

## Product Brief

`[Describe the product, users, and the main problem it solves.]`

## Technical Context

- Preferred stack: `[stack]`
- Deployment target: `[cloud/local/hybrid]`
- Expected scale: `[small/medium/high]`
- Data sensitivity: `[low/medium/high]`
- Compliance needs: `[none/GDPR/HIPAA/etc.]`

## Requirements

- Define the high-level system architecture.
- Identify services, modules, and boundaries.
- Explain the data flow.
- Recommend the folder structure.
- Call out security, observability, and testing concerns.
- Keep the design pragmatic and avoid unnecessary complexity.

## Output Format

Provide:

1. architecture overview
2. core modules and responsibilities
3. data model or storage strategy
4. API or interface boundaries
5. deployment outline
6. risks and tradeoffs
7. recommended next implementation steps
