# Refactor Prompt

Refactor the provided code without changing the intended behavior.

## Refactor Goals

- improve readability
- reduce duplication
- simplify control flow
- separate concerns
- improve testability
- preserve external behavior

## Constraints

- Keep naming clear and conventional.
- Prefer small focused functions.
- Remove dead code where safe.
- Do not introduce unnecessary dependencies.
- Note any assumptions that affect behavior.

## Output Format

Provide:

1. a short summary of the main problems
2. the refactored code
3. a brief explanation of why the new structure is better
4. tests that should be added or updated
